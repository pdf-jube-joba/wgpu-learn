3層ニューラルネットワークとレイトレーシングをやってみる。
球の書かれた画像から距離（カメラと球の中心）を推測する。

1. 空間の中に球を1つ置いてレイトレーシングをする
  - 左手系座標、上が z 
  - 3次元内で、球の半径は1に固定
    - 色は白の lambert 反射
    - 背景は黒
    - カメラは位置 (0,0,0) で、向きは (0,1,0) 、画角やスクリーンの大きさも位置も適当に定数にしておく
    - 光源はカメラから、光の強さは適当に、光は減衰しないものとする
  - 画面に一部は映っているようにするが、はみ出していてもいい
    - 画面に球の（スクリーン面に投影される面積のうちの）30%は映っていること
    - 画面の球が描画されない範囲は70%はあること
    - これを満たす画像が生成されれば分布は好きに決めてください。例えば、距離はガウス分布で決めて角度は一様分布とか、これがだめなら別の方法でも。この文章では定めないです。
  - N*N の白黒ピクセル画像でレイトレーシングをやる
    - 各ピクセルは f32 一つで白の強さを表し、0から1の範囲とする
2. 画像から距離を当てるのをニューラルネットワークで訓練する
  - 入力層、隠れ層、出力層
    - 隠れ層の幅は適当な定数に
    - 隠れ層の活性化関数は tanh を使う
    - 出力層は活性化関数無しの sum にして、正規化はなしのまま f32 とする。
  - 推測結果は距離なので実数になる
  - 誤差は平均二乗誤差
    - 画像を生成するときに距離を持ってきて、それを真の値とする
  - 訓練データとテストデータをわけない
    - 画像を乱数で生成して訓練データに使う
    - テストデータも乱数で生成したものを使う
3. プログラムについて
  - 必要になる定数っぽいものはプログラムに static で書く
  - 乱数はシードを固定してプログラムに書く
    - 乱数は使うごとに適当にシャッフル、ここはなんでもいい。
    - invocation ごとに必要な分は id を使って乱数をまぜる
  - プログラム実行時に引数はなしにする
  - 学習のやりかた:1step でやること
    - 何枚か画像を生成する
    - 順方向伝番では1invocationが1つのユニットを担当
    - 逆方向伝番でも1invocationでループで勾配を計算する
  - 学習結果は重みをそのままビット列にする
    - 定数だけあっていれば読めるように、あまり細かいことをしない
  - レイトレーシングも学習も compute shader を使う
  - 再現性はとくに考えなくていい
